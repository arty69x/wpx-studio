# UI Design Template

## UI Overview

Describe the product experience, supported devices, design system, navigation model, and primary user journeys.

## Design Tokens

Define colors, typography, spacing, radius, shadows, borders, motion, breakpoints, z-index layers, and density modes.

## Components

Document reusable components, variants, states, props, accessibility rules, content guidance, and examples.

## Screens

For every screen, describe purpose, route, entry conditions, layout, data dependencies, interactions, empty state, loading state, success state, error state, permission state, and responsive behavior.

## Forms

Define labels, validation, help text, error display, submission states, keyboard behavior, autocomplete, masking, and recovery.

## Accessibility

Specify semantic HTML, keyboard navigation, focus management, contrast, labels, announcements, target size, reduced motion, and screen reader behavior.

## Content Design

Define voice, terminology, capitalization, error messages, confirmation messages, destructive actions, and localization readiness.

## QA Notes

Define visual regression checks, responsive breakpoints, accessibility checks, browser matrix, and interaction edge cases.
